About

I build and automate AWS infrastructure, from IAM policy to CI/CD-ready image pipelines, with a security engineer's eye for what can go wrong.

I'm an IT Support Associate moving into cloud engineering, with hands-on experience across Linux, networking, and AWS. My focus is infrastructure as code: I use Terraform, Ansible, and Packer to turn manual, one-off configuration into repeatable, version-controlled systems, and I build automation that reacts to events instead of waiting for a person to notice them.

For every system I build, I document the before-state, the change, and the residual risk. Infrastructure is only trustworthy when the reasoning behind it is written down and reviewable, whether that's a Terraform plan, a runbook, or a pull request.

In Progress

Wazuh SIEM & XDR Homelab

A self-hosted Wazuh SIEM/XDR stack, deployed with Docker Compose and configured through Ansible. Agents on hardened Linux hosts provide file integrity monitoring, CIS security configuration assessment, and detection rules mapped to MITRE ATT&CK. Terraform provisions the AWS-side log delivery (S3 + SQS with scoped IAM) so the same SIEM ingests CloudTrail and GuardDuty from the aws-secure-baseline account: one correlation point for on-prem and cloud. The whole stack is defined and deployed as code rather than clicked together by hand, so standing it up again elsewhere is a rerun, not a rebuild. The write-up covers rule tuning, false-positive triage, and what I alert on versus suppress.

Hardened Linux Golden-Image Pipeline

A repeatable image-hardening pipeline: Packer builds the base image, Ansible applies CIS benchmark controls, and OpenSCAP scores compliance before and after. The pipeline itself is the point: a clean, versioned base image any team can build from, rather than a one-off box someone hardened by hand. The same Ansible role hardens EC2 in aws-secure-baseline, and Wazuh continuously verifies the result at runtime. Each control documents the before-risk, the change, and the residual risk, including the controls I deliberately skip and why.

Segmented Network & Intrusion Detection Lab

A segmented home network, with management, trusted, IoT, and quarantine zones enforced with VLANs and firewall policy, instrumented with Suricata for network intrusion detection. Suricata alerts feed the Wazuh SIEM for correlation, mirroring the VPC segmentation and flow-log analysis in the cloud projects. Designing the zones and policy from scratch is good practice for the same segmentation decisions that shape a VPC: what talks to what, and what shouldn't. The write-up documents the segmentation policy, the threat each boundary mitigates, and proof that key detections fire against generated test traffic.

Secure-by-Design AWS Baseline

A Terraform-deployed landing zone done right: least-privilege IAM, CloudTrail + Config + GuardDuty + Security Hub enabled, encrypted S3, and a segmented VPC. Ansible then hardens EC2 to CIS benchmarks. It's designed to be stood up in a new account with a single apply, which is the actual test of whether infrastructure as code is done right. Each step documents the before-risk, the change, and the residual risk.

Detection & Automated Remediation

An EventBridge and Lambda pipeline that reacts to findings. A public S3 bucket triggers auto-remediation, alongside detection rules for root-account usage, console logins from new locations, and security-group changes. The underlying pattern (an event triggers a Lambda, which takes action) generalizes to almost any operational response worth automating, not just security findings. The write-up explains which findings I auto-remediate, why, and the failure modes of getting it wrong.
